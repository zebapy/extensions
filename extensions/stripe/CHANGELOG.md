# Stripe Changelog

## [V2 Overhaul] - 2025-12-07

Major update introducing multi-account management, enhanced coupon workflows, and significantly improved user experience. This release maintains full backward compatibility with existing setups.

### Added

- **Multi-Account & Sandbox Management**
  - New "Manage Stripe Accounts" command for managing multiple Stripe profiles
  - Add, edit, and switch between multiple Stripe accounts or sandboxes
  - Support for Stripe's new Sandbox environments
  - Define separate profiles for test, live, or custom sandbox environments
  - Existing `stripeLiveApiKey` and `stripeTestApiKey` preferences automatically mapped to default profile

- **Coupon Management**
  - New "Create Coupon" command with support for value-based and percentage-based coupons
  - Expiration settings, custom naming, and direct clipboard copy
  - New "Create Quick Coupon" command for one-click percentage coupon creation with autogenerated IDs

- **Payment Link Creation**
  - New "Create Payment Link" command to generate payment links directly against Stripe products

- **Improved User Guidance**
  - Markdown-based inline guides for missing or invalid API keys
  - Updated README with full command reference, multi-account setup guide, and troubleshooting section

### Improved

- **Developer Experience**
  - Major codebase refactor with centralized management for shortcuts, mappers, color and icon renderers
  - Stronger type safety using official Stripe SDK types
  - Extensive JSDoc coverage throughout the codebase
  - Simplified utilities, hooks, and file structure for better maintainability

- **UX & Presentation**
  - Refined list-based commands (Payment Intents, Charges, Balance Transactions, Events, etc.)
  - Grouped and categorized data by relevance
  - Cleaner metadata display with improved readability
  - More intentional presentation of key fields

### Compatibility

- No breaking changes
- All existing API key preferences are fully supported
- Seamless upgrade path for users on earlier versions
